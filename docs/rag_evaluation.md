# RAG 평가결과

## 1. 평가 개요

본 평가는 `2026년 재난대응 AI 활용 현장교육 운영계획(안)` 문서를 대상으로 Naive RAG와 Advanced RAG의 검색 품질을 비교하기 위해 수행했다.

실제 API 키가 없는 환경에서도 구조 검증이 가능하도록 local fallback 검색을 제공한다. Supabase, OpenRouter, rerank API 키를 `.env`에 설정하면 동일한 코드가 실제 외부 API 기반 RAG 파이프라인으로 동작한다.

## 2. 평가 환경

| 구분 | 내용 |
| --- | --- |
| 문서 | 2026년 재난대응 AI 활용 현장교육 운영계획(안) |
| 청크 수 | 12개 |
| 청킹 방식 | 제목/조항 계층 보존 + 문단 기반 의미 청킹 |
| Vector DB | Supabase pgvector |
| Chat Model | OpenRouter |
| Re-Rank | Cohere rerank 또는 Jina reranker, local fallback |
| 평가 실행 | `npm run demo`, `POST /api/ask` |

## 4. 대표 실행 결과

질문: `비상연락망 점검 주기는?`

Advanced RAG local fallback 실행 결과, 최상위 근거로 `10. 비상연락망 및 운영 점검` 청크가 검색되었다.

근거 내용:

```text
비상연락망은 교육 전날과 교육 당일 시작 30분 전에 각각 1회 점검한다.
```

검색 결과:

| 순위 | 청크 | 평가 |
| --- | --- | --- |
| 1 | 10. 비상연락망 및 운영 점검 | 정답 근거 |
| 2 | 4. 활용 문서 범위 | 비상연락망 점검표 관련 보조 근거 |
| 3 | 3. 교육 개요 | 교육 운영 맥락 |

## 5. Naive RAG와 Advanced RAG 비교

| 항목 | Naive RAG | Advanced RAG |
| --- | --- | --- |
| 검색 질의 | 사용자 질문 그대로 사용 | 검색 친화 질의로 재작성 가능 |
| 검색 방식 | vector similarity 중심 | hybrid 검색 + rerank |
| 근거 선별 | 상위 청크 단순 사용 | 후보 청크 재정렬 후 근거 축소 |
| 장점 | 구조가 단순하고 빠름 | 질문 의도와 실제 근거 일치도가 높음 |
| 한계 | 유사하지만 답이 없는 청크가 섞일 수 있음 | 외부 rerank API 사용 시 비용과 지연 발생 |

## 6. 평가 결론

Advanced RAG는 질문과 직접 관련된 청크를 상위로 올리는 데 유리했다. 특히 공공기관 문서처럼 제목, 조항, 작성일자, 적용 범위가 중요한 자료에서는 제목 계층 metadata와 rerank 단계를 함께 사용하는 방식이 답변 근거를 설명하기 쉽다.

본 프로젝트는 API 키가 없는 환경에서도 local fallback으로 구조를 확인할 수 있으며, `.env`에 Supabase/OpenRouter/rerank API 키를 입력하면 실제 외부 API 기반 RAG 평가로 확장된다.
